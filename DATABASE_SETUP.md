# نظام قاعدة البيانات - دليل شامل

موقعك **يحتوي بالفعل** على نظام database متكامل يعمل بشكل تلقائي:

```
┌─────────────────────────────────────┐

│  - إضافة حالات جديدة                │
│  - تعديل الحالات                    │
│  - حذف الحالات                      │
│  - استيراد من JSON                  │
└─────────┬───────────────────────────┘
          │
          ├─────────────────────────────────────┐
          │                                     │
          ▼                                     ▼
    ┌─────────────┐                    ┌──────────────────┐
    │   Firestore │                    │  IndexedDB       │
    │  (السحابة)   │ ◄──────────────►  │ (التخزين المحلي) │
    │  النسخة الأساسية                │                  │
    └─────────────┘                    └──────────────────┘
          │
          │
          ▼
    ┌──────────────────────────────────┐
```

## 🗄️ المكونات الرئيسية

### 1. **Firestore Database (قاعدة البيانات الرئيسية)**

- **الموقع**: Firebase Console → Firestore Database
- **المجموعة**: `cases`
- **التخزين**: سحابي (متاح من أي مكان)
- **المميزات**:
  - نسخ احتياطية تلقائية
  - متزامن في الوقت الفعلي
  - آمن مع قواعد الأمان

### 2. **IndexedDB (التخزين المحلي)**

- **الموقع**: متصفح المستخدم (Local Storage)
- **الاسم**: `dentist_portfolio_v2`
- **المتجر**: `cases`
- **الفائدة**: 
  - عمل بدون انترنت (offline)
  - سرعة أسرع
  - نسخة احتياطية محلية

### 3. **Firebase Authentication (المصادقة)**

- **الإدارة**: Google Sign-In فقط
- **الإيميل المسموح**: `ficfgicgpxt44@gmail.com`
- **الأمان**: حماية كاملة

## 🔄 كيفية عمل البيانات

### عند إضافة حالة جديدة:

1. الإدارة تملأ النموذج في Dashboard
2. يتم حفظ البيانات تلقائياً في:
   - ✅ Firestore (النسخة الأساسية)
   - ✅ IndexedDB (النسخة المحلية)
3. Gallery تُحدّث تلقائياً
4. جميع الزوار يرون الحالة الجديدة فوراً

### عند تعديل حالة:

1. Admin يعدّل البيانات
2. يتم تحديث:
   - ✅ Firestore
   - ✅ IndexedDB
3. جميع الزوار يرون التحديث

### عند حذف حالة:

1. Admin يحذف الحالة
2. يتم الحذف من:
   - ✅ Firestore
   - ✅ IndexedDB
3. تختفي من Gallery فوراً

## 🚀 خطوات الاستخدام

### الخطوة 1: إعداد Firebase (مرة واحدة فقط)

```bash
1. اذهب إلى Firebase Console: https://console.firebase.google.com
2. انقر على مشروعك
3. ادهب إلى Firestore Database
4. تأكد من وجود مجموعة باسم "cases"
5. تحقق من قواعد الأمان:

rule match /cases/{document=**} {
  allow read: if true;  # الجميع يمكنهم القراءة
  allow write: if request.auth.uid != null && request.auth.token.email == 'ficfgicgpxt44@gmail.com';  # الإدارة فقط للكتابة
}
```

### الخطوة 2: إضافة حالة جديدة

```
1. ادخل الموقع
2. ابحث عن زر "إدارة" (عادة مخفي)
3. ادخل بـ Google Account (ficfgicgpxt44@gmail.com)
4. انقر "إضافة حالة جديدة"
5. امل البيانات:
   - العنوان
   - الفئة
   - الوصف
   - أضف الصور
6. انقر "حفظ الحالة"
7. الحالة تظهر تلقائياً في معرض الحالات
```

### الخطوة 3: عرض الحالات للزوار

- الحالات تظهر تلقائياً في `Gallery` section
- الزوار يمكنهم:
  - ✅ تصفح جميع الحالات
  - ✅ تصفية حسب الفئة
  - ✅ الضغط لرؤية التفاصيل
  - ✅ عرض جميع الصور

## 🆘 استكشاف الأخطاء

### المشكلة: لا ترى الحالات في معرض الحالات

**الحل**:
1. تحقق من اتصال الإنترنت
2. افتح Firebase Console
3. تأكد من وجود مجموعة "cases"
4. احدّث الصفحة (F5)

### المشكلة: لا يمكن إضافة حالة جديدة

**الحل**:
1. تأكد من تسجيل الدخول
2. تحقق من أن الإيميل = `ficfgicgpxt44@gmail.com`
3. حقق من قواعد أمان Firestore
4. تحقق من console للأخطاء

### المشكلة: تحميل الصور بطيء

**الحل**:
1. احضر الصور ذات حجم أقل
2. استخدم صيغة JPG بدلاً من PNG
3. تحقق من سرعة الإنترنت
